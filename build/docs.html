Documentation

Syntax

The syntax of Point is very similar to css. Line end indicators are necessary while within elements, like shown.

a {
 'Some text';
}

button {
 'Some other text';
}

an element is defined by its name, followed by its classes, id, attributes and content:

button#mybuttonid.mybuttonclass {
 onclick: 'button_click_handler()';
 'Click me';
}

statements are called by their name, their argument and a trailing semicolon:

attach 'Hello world!'

Constants

Constants are values that can be used throughout your code as data

// string
'Hello world!'
// number (this can be negative or decimal aswell)
10
// boolean
true
// null type
null

Logic

The supported operators of this language are < > - + / * = == & |
Strings can be modified/compared with == != +, numbers with < > - + / * and booleans with & |

Variables

Variables can be used to access constants multiple times and quickly throughout the code. You can assign to one like this:

$myvar = 'Hi!';

And then use it like this:

$myvar

Comments

Comments are useful for documenting your code or disabling functionality/lines of your code. Inline comments are written with a double-slash like this:

// This is a comment

It is convention to put a space after // to make it more readable and look nicer.

Importing

The import statement allows for quick file linking, based on the file type, the functionality differs. If the specified file is a css file, it will be linked like a stylesheet, this is also suppored for ico files as icons, html files as preloads, js files as script elements. The pt file extension during build, will be transpiled aswell, to the matching directory structure.

Attaching

The attach statement, during build will read the file's content and wrap it around the transpiled result, to delimit between above and below, use $&
